Talks API¶

You can send messages to other users when you have to. For this, we provide a talks API.

There are talks and messages. A talk contains messages, and a message is linked to a talk

Talks¶

Talks are also aften called “negociations”. Their goal are to initiate a discussion between two users: the authenticated one, and another one, specified by the first one.

A talk is always related to a Trip.

Validate¶

URL
/api/talks/id
Verb
POST
Data
validate: must be set to ‘True’ to have effect.

Validate the negociation talk. This deletes the talk, and create a temporary report

$ curl -X PUT http://localhost:8000/api/talks/18/ -d "validate=true"
OK

Delete¶

URL
/api/talks/id
Verb
PUT
Data
  • validate: must be set to ‘True’ to have effect.

Cancel the negotiation talk

Delete the negociation talk and associated messages, send a mail to the other user and redirect to the talk’s list.

Deleting a talk is only possible for one of the two users the talk is about:

$ curl -X PUT http://localhost:8000/api/talks/19/ -d "cancel=true&message='my+message'"

Create¶

Initiate the talk with another user:

$ curl -X POST http://localhost:8000/api/talks/ -d "trip_id=28&message='my+message'"
Created

New¶

URL
/api/talks/messages/
Verb
PUT
Data
  • trip_id: the related trip the user want to talk about

Send a message to the author of the trip announce. If the Talk does not exists, create it automatically:

$ curl -X POST http://localhost:8000/api/talks/22/messages/ -d "message='my+message+(2nd)'"
OK

Delete¶

It’s not possible to delete a single message. You can delete an entire talk, and, by the way, all related messages.
